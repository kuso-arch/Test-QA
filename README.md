## Пользовательские сценарии (Use Cases)

### Use Case 1: Приглашение нового клиента

**Цель:** Зарегистрировать нового клиента в системе.

**Основной сценарий:**
1. Пользователь переходит в раздел "Клиенты"
2. Нажимает кнопку "Пригласить клиента"
3. Заполняет поля: Имя, Email, Телефон
4. Нажимает кнопку "Пригласить"
5. Получает подтверждение об успешной отправке приглашения

**Альтернативный сценарий:** Заполнение только обязательных полей.

**Негативные сценарии (поле Телефон):**
- Ввод короткого номера (менее 10 цифр)
- Ввод очень длинного номера (более 15–20 цифр)
- Ввод номера только из цифр без формата
- Ввод номера с пробелами, тире, скобками
- Ввод номера со спецсимволами (+, *, #, ! и т.д.)
- Ввод номера с буквами (латиница и кириллица)
- Пустое поле телефона
- Ввод номера, начинающегося с нуля или без кода страны

**Негативные сценарии (поле Имя):**
- Ввод имени только из цифр
- Ввод имени только из спецсимволов (@, #, $, %, ^, &, *, (, ), -, +, = и т.д.)
- Ввод имени со смешением латиницы и кириллицы (два алфавита)
- Ввод очень длинного имени
- Пустое поле имени
- Ввод имени только из пробелов
- Ввод имени, состоящего из одного символа

**Негативные сценарии (поле Email):**
- Ввод email без символа @
- Ввод email с несколькими символами @
- Ввод email без доменной части
- Ввод email с пробелами
- Ввод email со спецсимволами в локальной части
- Ввод очень длинного email
- Пустое поле email
- Ввод некорректного домена

### Use Case 2: Изменение роли пользователя

**Цель:** Изменить уровень доступа существующему пользователю.

**Основной сценарий:**
1. Перейти в раздел "Пользователи"
2. Выбрать пользователя и открыть редактирование
3. Изменить роль
4. Сохранить изменения
5. Увидеть обновлённую роль в таблице

**Негативный сценарий:** После сохранения роль в таблице не обновляется.

---

### Use Case 3: Приглашение нового пользователя в систему

**Цель:** Добавить нового пользователя с определённой ролью.

**Основной сценарий:**
1. Пользователь переходит во вкладку **"Пользователи"**
2. Нажимает кнопку **"Пригласить"**
3. Заполняет поле Email
4. Выбирает нужную роль из выпадающего списка
5. Нажимает кнопку отправки приглашения
6. Получает подтверждение об успешном приглашении

**Альтернативный сценарий:** Заполнение только обязательных полей.

**Негативные сценарии:**
- Ввод некорректного email (спецсимволы, пробелы, только цифры, смешанный формат)
- Ввод email без домена
- Попытка отправить приглашение без выбора роли
- Ввод очень длинного email
